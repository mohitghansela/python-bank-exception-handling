While studying Python's Exception Handling, I wanted to move beyond theoretical examples and apply the concepts in a practical way.

To achieve that, I built a small Bank Management System that demonstrates how exceptions are used in real applications. The project includes custom exceptions, multiple exception handling techniques, logging, and structured error management.
📚 Concepts Covered
Concept	Description
try / except	Catching and handling runtime errors
else block	Executes only if no exception occurs
finally block	Always runs, useful for cleanup
raise	Manually throwing exceptions
Custom Exceptions	Creating user-defined exception classes
Multiple except blocks	Handling different error types separately
Exception Inheritance	Base class BankException with child exceptions
Logging module	Writing errors to a log file

🚀 Features

✅ Create bank accounts with validation

✅ Deposit and withdraw money with full error handling

✅ Transfer funds between accounts

✅ PIN verification with automatic lock after 3 incorrect attempts

✅ Transaction history and account statements

✅ Error logging using Python's logging module

🎯 Purpose of the Project

This project was built to demonstrate practical exception handling in Python using a realistic banking example.

It can be useful for:

Python learners

Students studying error handling

Developers practicing OOP + exception management

Teachers demonstrating exception handling concepts
